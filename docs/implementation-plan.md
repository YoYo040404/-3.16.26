# תוכנית יישום — שלב הבא

מסמך זה מתאר את שלבי העבודה הבאים בלבד (ללא יישום בקובץ זה).

## 1) שדרוג דף הבית
- חיזוק היררכיה: Hero ברור, ערך מרכזי, CTA ראשי בולט ל-WhatsApp.
- שיפור זרימת מסר: בעיה → פתרון → שירותים → הוכחות → אודות → יצירת קשר.
- התאמה למובייל תחילה, עם ריווח וקריאות RTL תקינים.

## 2) דפי שירות נפרדים
- יצירת דף נפרד לכל שירות מרכזי (פרקט, שטיחים, דקים, תיקונים).
- מבנה אחיד לכל דף: Hero, מתי צריך שירות, מה כלול, תהליך, FAQ, CTA.
- קישוריות ברורה בין דף הבית לדפי השירות.

## 3) מבנה נתונים משותף לשירותים
- הגדרת מקור נתונים מרוכז (למשל `lib/services.ts`) עם:
  - `slug`
  - כותרת ותיאור קצר
  - נקודות ערך
  - שאלות נפוצות לפי שירות
  - תמונות מומלצות
- שימוש במבנה אחיד לרינדור כרטיסים ודפים דינמיים.

## 4) מערכת CTA ל-WhatsApp
- שימוש גורף ב-`lib/contact.ts` לכל כפתורי/קישורי WhatsApp.
- יצירת טקסטים ייעודיים לפי הקשר (דף בית, שירות, גלריה, יצירת קשר).
- ודאות שכל קישור כולל `encodeURIComponent` לטקסט הודעה תקין.

## 5) SEO ומטא־דאטה
- הגדרת `title` ו-`description` ייעודיים לכל עמוד.
- Open Graph בסיסי לדפי הבית והשירותים.
- מבנה כותרות תקין (`h1` אחד לעמוד, היררכיית `h2`/`h3` ברורה).

## 6) גלריה ותמונות זמניות
- בניית מבנה גלריה עם לפני/אחרי + פרטי ביצוע.
- שילוב placeholders מסומנים בלבד: `[תמונה זמנית]` / `DEMO`.
- מעבר עתידי מסודר מנכסים זמניים לנכסים סופיים לפי `docs/asset-shot-list.md`.

## 7) סקציית אודות
- הצגת ג׳ריקו באופן אישי ואמין (ללא טענות לא מאומתות).
- הדגשת עובדות עסקיות מאושרות: ניסיון, אחריות, אזור שירות, עבודה אישית.
- חיזוק אמון עם מסר ברור על תהליך עבודה נקי ומדויק.

## 8) FAQ
- בניית שאלות נפוצות ממוקדות החלטה:
  - האם מתקינים בלבד?
  - מי קונה חומרים?
  - האם מבצעים תיקונים קטנים?
  - תוך כמה זמן חוזרים?
- תשובות קצרות, מדויקות, ללא הבטחות מופרזות.

## 9) סקציית יצירת קשר
- CTA ראשי ל-WhatsApp + טלפון תצוגה ברור.
- ניסוח מיקרו-קופי שמבקש לשלוח תמונות + עיר.
- נראות גבוהה במובייל ובסוף כל דף מרכזי.

## 10) QA מובייל RTL
- בדיקת שבירות במסכים צרים לכל הסקשנים המרכזיים.
- בדיקת כיווניות RTL: יישור, אייקונים, מרווחים, תנועת רכיבים.
- בדיקת נגישות בסיסית: ניגודיות, גדלי טקסט, פוקוס למקלדת.

## 11) אימות סופי: Lint/Build
- הרצה מלאה של:
  - `npm run lint`
  - `npm run build`
- טיפול רק בשגיאות/אזהרות רלוונטיות לשינויים שבוצעו בשלב היישום.
